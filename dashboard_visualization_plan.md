# sample-data.csv ベースのダッシュボード可視化案（Streamlit）

## 1. データ概要
- レコード数: 300件
- 期間: 2023-01-04 〜 2024-12-31
- 項目: `顧客ID`, `年齢`, `性別`, `地域`, `購入カテゴリー`, `購入金額`, `購入日`, `支払方法`

## 2. まず見るべき主要KPI（画面上部）
- 総売上: **5,748,734**
- 平均購入単価: **19,162**
- 取引件数: **300**
- 期間内の月平均売上: `総売上 / 月数`
- 家電カテゴリ比率: **68.0%**（売上の偏りを即時把握するため）

## 3. フィルタ設計（サイドバー）
- 日付範囲（`購入日`）
- 地域（関東 / 関西 / 中部 / 九州）
- 購入カテゴリー（家電 / スポーツ / ファッション / 食品 / 書籍）
- 性別
- 支払方法（クレジットカード / 電子マネー / 現金）
- 年齢レンジ（スライダー）

## 4. 可視化レイアウト案

### セクションA: 全体トレンド
1. **月次売上推移（折れ線グラフ）**
   - 目的: 季節性・繁閑の把握
   - 補足: 2024-07 がピーク（409,432）
2. **月次件数推移（棒グラフ）**
   - 目的: 売上増が「客数増」か「単価増」かを切り分ける

### セクションB: 売上構成
1. **カテゴリ別売上（横棒グラフ）**
   - 目的: 主力カテゴリの把握
   - 補足: 家電が売上の 68.0% と支配的
2. **地域別売上（棒グラフ）**
   - 目的: エリアごとの優先度判断
   - 補足: 関西 29.0%、中部 25.5%、九州 24.0%、関東 21.4%
3. **支払方法の利用比率（ドーナツ）**
   - 目的: 決済導線の最適化
   - 補足: クレカ 127件、電子マネー 109件、現金 64件

### セクションC: 顧客セグメント
1. **年代別売上（棒グラフ）**
   - 目的: 主要ターゲット層の確認
   - 補足: 60代の売上合計が最大（1,149,024）
2. **年代別平均購入単価（棒グラフ）**
   - 目的: 高単価層の特定
   - 補足: 70代以上の平均単価が高い（26,611）
3. **性別 × カテゴリ（ヒートマップ）**
   - 目的: 施策対象セグメントを具体化

### セクションD: 深掘り分析
1. **地域 × カテゴリの売上ヒートマップ**
   - 目的: 地域特性に合う商品戦略の判断
2. **散布図（年齢 vs 購入金額、色=カテゴリ）**
   - 目的: 年齢と購入金額の関係を探索
3. **詳細テーブル（フィルタ連動）**
   - 目的: ドリルダウン確認、CSVダウンロード対応

## 5. Streamlit 実装時のページ構成例
- `st.set_page_config(layout="wide")`
- 1段目: KPIカード（`st.columns(5)`）
- 2段目: 月次トレンド + 月次件数
- 3段目: カテゴリ別売上 + 地域別売上 + 支払方法比率
- 4段目: 年代別売上 + 年代別平均単価 + 性別×カテゴリ
- 5段目: 地域×カテゴリヒートマップ + 散布図
- 最下部: 明細テーブル + ダウンロードボタン

## 6. 初期リリースで特に有効なインサイト例
- 売上の多くが家電に集中しているため、カテゴリ依存リスクをモニタリングする。
- 関西の売上寄与が最も高く、地域別キャンペーンの優先候補になる。
- 60代・70代の売上/単価が高く、シニア層向け施策の検証価値が高い。
- 現金比率が低いため、キャッシュレス施策の効果測定がしやすい。

## 7. 追加で入れると便利な機能
- 前年同月比（YoY）トグル
- 売上上位Nカテゴリ切替
- 閾値アラート（例: 月次売上が20万円を下回ったら注意表示）
- フィルタ条件をURLクエリに保持（共有しやすくする）
